# Ekzo-Shop
IdealShop for ppl
## 🗄 Структура базы данных (ТЗ на модели)

### 1. Пользователи (App: `users`)
*   **User:** расширенная модель.
    *   `username / email` — авторизация.
    *   `phone` — связь.
    *   `health_notes` — аллергии и особенности здоровья (TextField).

### 2. Каталог (App: `catalog`)
*   **Category:** группы трав.
    *   `name` — (Успокаивающие, Иммунитет и т.д.).
    *   `slug` — понятный URL.
*   **Herb:** карточка товара.
    *   `title` — название (рус).
    *   `latin_name` — название (латынь).
    *   `description` — описание.
    *   `indications` — показания к применению.
    *   `contraindications` — **противопоказания** (обязательно).
    *   `preparation_method` — как заваривать/готовить.
    *   `price` — стоимость.
    *   `stock` — остаток на складе.

### 3. Заказы (App: `orders`)
*   **Order:** общая информация о покупке.
    *   `user` — кто заказал.
    *   `status` — (Новый / Оплачен / Отправлен / Доставлен).
    *   `total_price` — итоговая сумма.
*   **OrderItem:** детализация (состав заказа).
    *   `order` — связь с заказом.
    *   `herb` — какую траву купили.
    *   `quantity` — сколько (вес/пачки).
    *   `price_at_purchase` — фиксация цены на момент покупки.

### 4. Оплата (App: `payments`)
*   **Payment:** данные транзакции.
    *   `order` — ссылка на заказ.
    *   `payment_status` — (Ожидает / Успех / Ошибка).
    *   `transaction_id` — ID из системы банка.
    *   `bank_response` — технический лог ответа банка.
