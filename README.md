# Ekzo-Shop
IdealShop for ppl

## 🖼 Техническое задание: Frontend (Лавка трав)

### 1. Архитектура и стек
*   **Фреймворк:** React / Vue.js (на выбор).
*   **Стейт-менеджер:** Redux Toolkit / Pinia (для корзины и данных пользователя).
*   **Стилизация:** Tailwind CSS / SCSS (для быстрой и адаптивной верстки).
*   **Запросы:** Axios (настроенный экземпляр с базовым URL к Django API).

### 2. Основные страницы (Views)
*   **Главная:** Баннер, популярные категории и блок «Трава дня».
*   **Каталог:** Сетка товаров с фильтрацией по категориям и поиском по названию/латыни.
*   **Карточка товара:** 
    *   Подробное описание и способ приготовления.
    *   **⚠️ Внимание:** Блок «Противопоказания» выделить красным акцентом.
*   **Корзина:** Список выбранных трав, изменение веса/количества, расчет итоговой суммы.
*   **Личный кабинет:** История заказов и поле «Заметки о здоровье».
*   **Оформление заказа:** Форма ввода телефона, адреса и имитация кнопки «Оплатить».

### 3. Функциональные требования
*   **Работа с API:**
    *   Получение списка трав (`GET /api/catalog/herbs/`).
    *   Регистрация и JWT-авторизация.
    *   Отправка заказа (`POST /api/orders/`).
*   **Валидация:** Проверка полей телефона и обязательного согласия с тем, что пользователь ознакомлен с противопоказаниями.
*   **Адаптивность:** Сайт должен удобно открываться с телефона (травники часто ищут информацию «в полях»).

### 4. Интеграция с бэкендом (Mocking)
*   На время разработки бэкенда использовать временные JSON-данные (Mock data), соответствующие структуре моделей в ТЗ.

### 5. Приоритеты (MVP)
1.  Верстка каталога и карточки товара.
2.  Логика корзины (локальное хранилище).
3.  Интеграция с API (авторизация и заказы).
